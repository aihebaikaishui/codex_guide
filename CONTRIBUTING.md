# Contributing

感谢你愿意一起维护这个仓库。

## 你可以做什么

- 修正文档错别字或表达不清的地方
- 补充 `project_brief`、`project_status` 的模板示例
- 优化目录结构和文档导航
- 提出更适合开源协作的规范

## 提交建议

1. 先打开相关文档确认上下文。
2. 尽量保持中文表达清晰、简洁、可复用。
3. 如果新增术语，请在首次出现时解释一下。
4. 如果会影响原有结构，请同步更新 `README.md`。

## 变更原则

- 小步修改，尽量只改必要部分。
- 如果是示例内容，尽量写成可以直接复用的模板。
- 不要引入仓库当前不需要的复杂依赖。

## 提交说明

建议在 PR 描述里说明：

- 修改了哪些文件
- 为什么要改
- 是否需要同步更新示例或模板
